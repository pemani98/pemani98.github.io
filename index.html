Work Experiences

Flatiron Health

Security/Software Engineer Summer 2019

The past summer I supported the infrastructure security team by creating an alert system that cautioned the team members of users with extended resource access and other privileges on our AWS cloud environment. This alert system actually ended up being implented on Splunk. First, I used Cloudtracker,a third party tool created by Duo, to write a script that catches the actions the users and/or roles in an AWS accounts have not taken in a given time limit. From that, I derived more information about that action including which policy it came from, the resource it affected, the service it involved, and some more information about the user/role as well. I pushed all of this into a sqlite database using sql alchemy and further migrated it into a csv file. This csv file was configured to push its contents into a splunk dashboard, so I could create a Splunk alert.

Lockheed Martin

Cyber Intern Summer 2018

At Lockheed Martin, I had the unique opportunity to support a team that built helicopters, in fact, they built the Presidential helicopter. In 2015, Lockheed Martin incoporated this company, Sikorsky, into their own organization as both companies built air-crafts. I worked to secure the avionics interface of a speific helicopter that carried cargo, weapons, and refugees to and from the United States and other countries. A rather huge concern these days is remotely hacking into an aircrafts interafce and navigating it from fruther way. So I worked with RBAC, role based access control, functionalities to breach these vulnerabilities. In addition to this, my team had me do some vulnerability analysis of scans they took of this air-craft. I also worked with two other interns to recommend more robust penetration testing tools. Of the various different types of tools, I focused on packet crafter. Trex and Ostinato are two packet crafters I found to be most effective in terms of functionality and flexibility, but Trex was more applicable to aircraft interfaces, so that was my conclusion. A final project I did with other interns was build a robot that tracked faces using facial recognition algorithms. We used an arduino to power servos that were connected to a webcam. We used Processing to help the webcam identify faces, and if the webcam did not, Processing would communicate this with the arduino, and in response, the arduino would move servos until the webcam caught faces.
